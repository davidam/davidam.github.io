1 Instalación de Paquetes

Una manera rápida para hacerlo con mysql puede ser la siguiente: 

sudo apt-get install drupal7

Drush es una herramienta útil para facilitar la administración de nuestra línea de comandos, por ejemplo, actualizaciones de módulos, copias de seguridad, etc.

Realmente, es mejor descargar la última versión de drupal en vez de utilizar el código dado por el paquete debian de drupal, debido mi política de usar drush para actualizaciones. Pero instalar drupal7 como paquete debian es una buena idea para instalar dependencias. 

$ cd /var/www/
$ sudo wget -c http://ftp.drupal.org/files/projects/drupal-7.32.tar.gz
$ sudo tar -xvzf drupal-7.32.tar.gz
$ sudo chown -R www-data.www-data drupal-7.32
$ sudo chmod g+w -R drupal-7.32
$ sudo mv drupal-7.32 mysite

2 Configurar Apache

Al instalar un sitio drupal, me gusta habilitar urls bonitas, para ello creo el fichero mysite en /etc/apache2/sites-available/mysite, con el siguiente contenido: 

<VirtualHost *:80>

  ServerAdmin webmaster@localhost
  ServerName mysite

  DocumentRoot /var/www/mysite

  <Directory /var/www/mysite>
                RewriteEngine on
                RewriteBase /mysite
                RewriteCond %{REQUEST_FILENAME} !-f
                RewriteCond %{REQUEST_FILENAME} !-d
                RewriteRule ^(.*)$ index.php?q=$1 [L,QSA]
  </Directory>

</VirtualHost>

Puesto que en un principio igual no tendremos el dominio del sitio se puede habilitar el host en /etc/hosts 

127.0.0.1       localhost
127.0.1.1       mysite

Después se puede habilitar de la siguiente manera: 

$ sudo a2ensite mysite

3 Creación de la base de datos en Mysql

$ mysql -u root -p 
mysql> create database mysite;
mysql> create database drupal7;
mysql> create user 'drupal7'@'localhost' identified by 'lapasswd';
mysql> grant all privileges on drupal7.* to 'drupal7'@'%' with grant option;

4 Creación de la base de datos en postgresql

Primero, instalar los paquetes debian: 

$ sudo apt-get install postgresql-9.1 postgresql-client php5-pgsql

Después, crear la base de datos y el usuario a postgres. 

$ createuser dpsql --pwprompt --encrypted
$ createdb dpsql --owner=dpsql

Para usar el cliente psql en local se puede modificar la línea local en /etc/postgresql/9.1/main/pghba.conf: 

local             all            all                   trust

Si se quiere cambiar la contraseña se puede hacer: 

$ psql -U dpsql dpsql -W
dpsql=> ALTER USER dpsql with password 'thepassword';

5 Instalación Web

Una vez habilitado el sitio se puede ir a http://mysite/install.php y la instalación es seguir un paso a paso.

Primero, se pueden eliminar permisos a CHANGELOG.txt para evitar problemas de seguridad: 

$ cd /var/www/mysite/
$ chmod 600 CHANGELOG.txt

Así habilitado el sitio, se puede ir a http://mysite/install.php y la instalación es hecha paso a paso.

En el paso 4 (Configurar la base de datos) se puede encontrar el nombre de la base de datos, username de la base de datos y contraseña de la base de datos con los datos provistos.
